# DLP Exception Governance Agent

> **Microsoft Copilot Studio** agent for reviewing, prioritizing and updating DLP exceptions by combining the exception inventory with real DLP alert activity.

---

## What does the agent do?

DLP exceptions are necessary for some business processes, but they become a risk if they are not reviewed. This agent combines two data sources to provide clear and actionable answers:

```
Exception inventory  +  DLP alerts export
              ↓
     Automatic correlation
              ↓
  Risk scoring  +  Recommendation  +  Inventory update
```

---

## Data sources

| File                         | Description                                       |
| ---------------------------- | ------------------------------------------------- |
| `DLP_Exceptions_MVP_EN.xlsx` | Inventory of DLP exception requests and approvals |
| `DLP_Alerts_MVP_EN.xlsx`     | Periodic export of alerts generated by DLP rules  |

---

## Main capabilities

| Capability                  | Description                                                     |
| --------------------------- | --------------------------------------------------------------- |
| 🔍 Review active exceptions | Identifies which exceptions are still valid                     |
| ⏰ Detect expired exceptions | Finds expired exceptions that require action                    |
| 📊 Calculate recent alerts  | Counts associated alerts from the last 30 days                  |
| ⚡ Prioritize by activity    | Highlights exceptions with the highest recent activity          |
| 🚫 Detect unused exceptions | Finds active exceptions with no recent activity                 |
| 🧮 Assess risk              | Applies scoring based on status, expiry, activity and data type |
| ✅ Recommend actions         | Suggests renewing, reviewing, removing or escalating exceptions |
| 📝 Update the inventory     | Completes risk and activity fields in the Excel inventory       |

---

## Correlation logic

The agent correlates both datasets using three fields:

```
Requester  =  User
DLPPolicy  =  Policy
Date within the last 30 days
```

Using this correlation, it calculates `AlertsLast30Days` and generates the governance fields:

| Field               | Description                                    |
| ------------------- | ---------------------------------------------- |
| `AlertsLast30Days`  | Number of alerts associated with the exception |
| `LastActivityDate`  | Most recent activity date found                |
| `RiskScore`         | Numeric result of the scoring model            |
| `RiskLevel`         | Low · Medium · High · Critical                 |
| `RecommendedAction` | Renew · Review · Remove · Escalate             |

## MVP scope

**Included**

* Synthetic dataset of DLP exceptions and alerts
* Correlation by user, DLP policy and date
* Calculation of alerts from the last 30 days
* Basic risk scoring and governance recommendations
* Assisted update of the exception inventory

**Out of scope**

* Direct connection with Microsoft Purview
* Use of real APIs or a Dataverse data model
* Automatic remediation without human review

---

> ⚠️ The data included in this repository is **synthetic**. It must not include real user, customer or production environment data.
